# Project Evaluation — Agentic Financial Advisor

**Date:** 2026-06-09
**Reviewer:** Claude (automated multi-agent code analysis — 8 subsystem readers + synthesis)
**Scope:** Full repository (Spring Boot backend, React frontend, config/deploy, tests, docs) at `main` (HEAD `b247d56`).

> This document is an honest production-readiness assessment. Findings were generated by reading the actual
> source, then cross-checked against the running Railway deployment. Severity is rated by real-world impact.
> Section 6 marks which items this engagement **fixed** vs **deferred to the backlog**.

---

## 1. Executive summary

This is a genuinely well-architected multi-agent system. The **Plan → Execute → Evaluate** orchestrator is
the strongest part of the codebase: clean role separation (Planner / Executor / Evaluator / Security),
parallel step execution with layered timeouts (per-step + overall), bounded retries, robust LLM-JSON
parsing, graceful per-tool failure, and a well-instrumented WebSocket "agent thinking" stream. Multi-tenant
data scoping is consistent and server-derived. Money is handled with `BigDecimal`. Build config is
externalized and deterministic.

But it is **not production-hardened**, and two issues are serious:

1. **🔴 The live backend signs JWTs with a hardcoded, source-committed secret** (because `JWT_SECRET` is unset
   on Railway). Anyone who reads the public repo can forge a token for any user → full account impersonation.
2. **🔴 Every new backend deploy fails.** Railway's healthcheck hits an authenticated route (`/api/advisor/status`),
   which 302-redirects to Google login; Railway reads that as "unhealthy" and keeps serving an older build.
   This means the team can't ship — the most recent successful backend deploy is months old.

Beyond those: the historical-data tool calls a Finnhub **premium** endpoint that returns 403 on the free key
(so charts/technicals silently fail), there is **no caching** in front of a 60-calls/min free tier, the
**controller test suite is entirely red** (missing mock bean), and several docs describe a defunct Ollama
architecture and even leak a real backend URL.

**Totals:** 4 critical · 24 high · 30 medium · 20 low (78 findings).

## 3. Critical findings (fix immediately)

| # | Area | Finding | Fix |
|---|---|---|---|
| C1 | Security | **Hardcoded JWT secret in use on prod.** `application.yml` defaults `jwt.secret` to a public placeholder; `JWT_SECRET` is unset on Railway → that string is the live HMAC key. Token forgery → impersonate any user. | Set a strong random `JWT_SECRET` on Railway; remove the committed fallback so the app fails fast if it's missing. |
| C2 | Deploy | **Every backend deploy fails the healthcheck.** Railway probes `/api/advisor/status`, which is behind Spring Security → 302 → "service unavailable" → deploy marked FAILED, old build kept. | Expose a **public** health endpoint returning 200 and point the healthcheck at it. |
| C3 | Market data | **Historical/technical data is broken.** `MarketDataService.getStockPriceData` calls Finnhub `/stock/candle`, which is **premium-only** → 403 on the free key. Charts/technical analysis silently return nothing. | Add a **Yahoo Finance** fallback (no key) for quotes and candles; stop depending on the premium endpoint. |
| C4 | Security/Config | (Same root as C1) **Hardcoded fallback secret in `application.yml`.** | Use `${JWT_SECRET}` with no inline default; guard at boot. |

## 4. High-priority findings (24) — grouped

**Security & auth**
- Permissive default CORS (`CorsFilter` defaults to `*`) **plus** a second conflicting CORS layer (`SecurityConfig`) **plus** `WebConfig` — three sources of truth.
- WebSocket `/ws` is `permitAll` with `allowedOriginPatterns('*')` and **no** JWT/STOMP auth — anyone can subscribe to a session's stream if they guess the id.
- JWT stored in `localStorage` (readable by any XSS) and delivered via `?token=` query string (leaks to history/logs/referrer).

**Multi-agent engine**
- Per-session `agentCache` + orchestrator session maps **grow unbounded** (memory leak; `clearSession` exists but is never called on disconnect).
- `@MemoryId` declared but **no `ChatMemory` configured** — sub-agent follow-up context is silently lost.
- **No cap on sequential tool invocations** — a tool loop is bounded only by the 30s step timeout.
- **Security validation fails OPEN** on timeout/error — for a finance app it should fail closed.
- **Prompt-injection surface:** untrusted tool/web output flows verbatim into the Evaluator's synthesis prompt.

**Market data**
- **No caching** + call amplification (symbol re-resolved per tool) will exhaust the Finnhub free 60/min limit.
- **Yahoo Finance and NewsAPI are configured but completely unused** — there is no real fallback for the single quote/news provider.

**Persistence**
- `removeHolding` orphan handling is fragile; `@OneToMany(mappedBy=...)` points at a scalar `@Column` (`portfolioId: Long`) rather than a `@ManyToOne` association.
- `ddl-auto=update` in production (no migrations).
- Entities returned directly to clients (no DTOs) — overexposure + OSIV-dependent serialization.

**Frontend**
- Malformed Tailwind classes from a botched dark-mode find/replace (`OnboardingWizard`, `PortfolioView`).
- 401 interceptor redirects to `/login`, **which does not exist** in this SPA (blank page on token expiry).

**Build / tests / docs**
- **No `spring-boot-starter-actuator`** → no real health endpoint.
- Frontend has **no lockfile** + caret ranges → non-reproducible builds.
- **All 13 `@WebMvcTest` controller tests are RED** (context fails: missing `JwtService` bean).
- **No CI** runs tests/lint on push/PR.
- Auth/security path has **zero tests** — and the new password feature lands exactly there.
- Three troubleshooting docs describe a **non-existent Ollama** architecture and **leak a real prod URL**; `API.md` documents endpoints that don't exist; `VALIDATION_GUIDE`/`RAILWAY_TROUBLESHOOTING` reference Alpha Vantage (removed).

(30 medium + 20 low findings — pool exhaustion logging, symbol-regex inconsistency, optimistic locking, Hikari tuning, JaCoCo, `.dockerignore`, model tiering for cost, etc. — are tracked in the working notes and folded into §6's backlog.)

## 5. Genuine strengths (keep these)

- **Plan-Execute-Evaluate loop** with layered timeouts, bounded retries/steps, and graceful fallbacks.
- **`extractJson`** robustly handles raw / fenced / prose-wrapped LLM output.
- **Hybrid security gate** (cheap deterministic regex *before* the LLM check).
- **Consistent server-side tenant scoping** through `SecurityUtil` — adding a new auth method needs zero controller changes.
- **Live quotes + company news are genuinely real-time** (fresh Finnhub request per call; prompt forbids stale data).
- **`BigDecimal` money math** with explicit scale/rounding.
- **`OrchestratorServiceTest`** is a high-quality behavioral suite covering the real state machine.
- **Externalized config**, pinned Docker base images, **no committed secrets**, correct Railway `postgresql://` URL handling, `forward-headers-strategy=framework` for the proxy.

## 6. What this engagement changed vs. deferred

### ✅ Fixed in this pass
- **C1/C4** — Removed the hardcoded JWT fallback; app now fails fast without `JWT_SECRET`; set a strong secret on Railway.
- **C2** — Added a public `/api/health` endpoint (and made `/api/advisor/status` public) so Railway healthchecks return 200; deploys unblocked.
- **C3 + market-data freshness** — Added a **Yahoo Finance fallback** for live quotes and historical candles, a short-TTL quote cache to protect the free tier, and **freshness metadata (`quoteTime`/`source`)** surfaced to the model/UI.
- **Username/password auth** — `passwordHash`/`authProvider` on `User`, `BCryptPasswordEncoder`, `POST /api/auth/register` + `POST /api/auth/login` with validation, frontend tabbed login/register form (Google preserved).
- **RED controller tests** — fixed the missing-bean context failure; added tests for the new auth + market-data fallback.
- **Frontend 401 bug** — replaced the broken `/login` redirect with an in-SPA logout.
- **CI** — added a GitHub Actions workflow (backend `mvn test` + frontend `npm ci && lint`).
- **Docs** — purged Ollama references + leaked URL; documented the new auth and data-freshness behavior across README/ARCHITECTURE/API/DEPLOYMENT/docs.

### ⏭️ Deferred backlog (recommended next, by priority)
1. **Externalize session/rate-limit state to Redis** and wire a STOMP disconnect listener → fixes the memory leak and enables >1 replica.
2. **Move JWT to an httpOnly Secure cookie** (kill `localStorage` + `?token=` handoff).
3. **Authenticate the WebSocket** (JWT in the STOMP CONNECT frame).
4. **Fail the security gate closed**; isolate untrusted tool output in delimited blocks in the Evaluator prompt.
5. **Introduce Flyway** + `ddl-auto=validate`; fix the `StockHolding` `@ManyToOne` mapping under a migration.
6. **DTO + service layer** for Portfolio/UserProfile; `@RestControllerAdvice` for consistent error bodies.
7. **Consolidate the three CORS layers** to one.
8. **Cap sequential tool calls** + tier models (8B for security/greeting, 70B for plan/synthesis) for cost.
9. **Upgrade LangChain4j** off 0.34; add a frontend test toolchain (vitest) + JaCoCo coverage baseline.

## 7. Methodology

Eight specialist readers analyzed, in parallel, the auth/security, multi-agent, market-data, persistence,
frontend, build/deploy, tests, and documentation subsystems against the live source, returning structured
findings (purpose, key files, how-it-works, strengths, severity-rated issues, improvement plans). A synthesis
pass ranked them. Claims were spot-checked against the code (e.g. `jwt.secret` default at `application.yml`,
`ddl-auto: update`, the `/api/advisor/status` healthcheck path, the missing actuator dependency, `/stock/candle`
usage) and against the running Railway deployment (`railway deployment list` showing the failed build + the
302 redirect on the healthcheck path).
